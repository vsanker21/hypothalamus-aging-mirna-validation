STATISTICAL ANALYSIS PLAN (repository scope) — pre-specification for MOESM + public anchors
Last updated: 2026-05-15

1) PRIMARY ANALYSIS FAMILIES (pre-specified for this computational repo)
A. Zhang MOESM (run_pipeline.py)
   - Primary: descriptive summaries and planned plots for miRNA, exosome/nanoparticle, cytokine, Sox2, and phenotype tables as loaded from MOESM spreadsheets.
   - Multiplicity: where many miRNAs are scanned, use the implemented permutation or FDR strategy documented in outputs/REPORT_SUMMARY.txt (do not silently threshold-shop).
B. Extended integration (run_extended.py)
   - Primary: miRTarBase target union + Hallmark ORA + preranked GSEA (target multiplicity) + HypoMap Fisher (with contrast disclosure) + optional GSE188646 Fisher at padj<=0.05 for DE ∩ miRTarBase universe.
   - Exploratory (pre-labelled outputs): GSE188646 padj<=0.1 Fisher; top-200 / top-500 rank-score DE gene sets; pseudobulk Hallmark fgsea (multilevel when available) on DE ranks; limma camera and fry on pseudobulk log-CPM; DoRothEA TF activity (mlm) on pseudobulk log-CPM; two-cohort DL meta (GSE188646 + GSE87102) by default when cohort1 DE exists (opt-out GSE188646_SKIP_COHORT2_META=1); multi-library ORA with global BH; bootstrap target stability.

2) MULTIPLICITY CONTROL POLICY
- Primary MOESM claims must reference the pipeline’s stated correction or permutation for high-dimensional miRNA scans.
- Exploratory arms (exploratory_* files, multi-library ORA, rank-based DE sets) must be reported as sensitivity / hypothesis generation — not upgraded to primary endpoints post hoc.
- Do not combine unrelated Fisher tests (HypoMap vs GSE188646) as confirmatory meta-evidence.

3) POWER / DESIGN HONESTY
- GSE188646 pseudobulk: 4 Young vs 4 Aged biological replicates (orig.ident) — adequate only for large, consistent pseudobulk effects; subtle cell-type–mixed programs may not reach FDR in bulked pseudobulk.
- Interaction models (e.g. LIFU × age × cell type) are not supported by default in this repo — require dedicated experimental N and pre-registration.

4) ENDPOINT MAPPING (for grants / IRB text)
- Molecular panel: MOESM miRNA + exosome miRNA tables.
- Tissue homeostasis panel: Sox2 / ARC counts where present.
- Inflammation / secretome panel: cytokine tables where present.
- Public anchor panel: GSE188646 + HypoMap + GenAge — contextual only unless contrast-matched replication is added (see REPLICATION_AND_META_POLICY.txt).

5) VERSIONING
- Record Python + R package versions whenever outputs are frozen for publication (tools/snapshot_versions.py → outputs/VERSION_SNAPSHOT.txt).
